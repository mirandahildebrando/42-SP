42 São Paulo - Piscina C (Fevereiro/2026) 🏊‍♂️
Este repositório documenta minha jornada intensiva de 26 dias na 42 São Paulo, um ambiente de alto rendimento focado em engenharia de software e aprendizado autodidata.

🎯 Objetivo
Desenvolver fundamentos sólidos de computação de baixo nível e lógica de programação avançada, com foco total na Linguagem C e sistemas Unix-like.

🛠️ Stack Técnica & Palavras-Chave
Linguagem Principal: C (C99/ANSI).

Gestão de Memória: Alocação dinâmica (malloc, free), manipulação de pointers e prevenção de memory leaks.

Algoritmos: Ordenação (sorting), recursividade, manipulação de strings e busca.

Ambiente: Terminal Linux (Bash/Zsh), permissões de arquivos, processos e pipelines.

Compilação & Debugging: gcc, cc, flags de erro estritas (-Wall -Wextra -Werror).

Versionamento: Git (Version Control).

🚀 Competências Desenvolvidas
1. Rigor e Padronização (Clean Code)
Todo o código foi escrito seguindo a Norma da 42, um padrão de estilo extremamente rigoroso que exige funções curtas, nomes de variáveis semânticos e proibição de bibliotecas externas padrão, forçando a recriação de funções do zero (ex: ft_strlen, ft_strcpy).

2. Resolução de Problemas Sob Pressão
A metodologia da 42 foca no "aprender a aprender". Resolvi desafios lógicos complexos sem auxílio de professores, utilizando apenas documentação oficial, testes de unidade e colaboração entre pares (Peer-to-Peer Learning).

3. Engenharia de Baixo Nível
Compreensão profunda de como a memória do computador funciona, como os dados são organizados em stacks e heaps, e como otimizar o uso de recursos do sistema.

4. Resiliência e Autodidatismo
Capacidade de absorver novas tecnologias em ciclos de 24 horas, entregando projetos funcionais e validados por sistemas de correção automática e avaliações presenciais.
